# TPC-DS HTML Report Design

## Цель

Сформировать один автономный HTML-файл по всем 99 официальным запросам TPC-DS, отсортированный от наиболее структурно сложных к наиболее простым. Отчёт должен позволять быстро увидеть сложность корпуса и понять, какие части каждого SQL-запроса распознал SQLGlot.

## Источник данных

- Закреплённая версия запросов: DuckDB commit `9ebdd1ee5279885dd2a89d4ac8f37034c05de203`.
- Каталог на входе содержит `01.sql`–`99.sql`.
- Диалект разбора по умолчанию: `postgres`, совместимый с целевым Greenplum 6 для синтаксического анализа.
- HTML не обращается в интернет и содержит весь текст запросов внутри файла.

## Определение сложности

Рейтинг отражает структурную сложность AST, а не стоимость выполнения. Для оценки не нужны таблицы с данными, статистика Greenplum или `EXPLAIN`.

Для каждого файла вычисляется:

```text
score = round(
    0.1 * AST nodes
  + 4.0 * CTE
  + 6.0 * subqueries
  + 6.0 * set operations
  + 2.0 * joins
  + 4.0 * windows
  + 2.0 * CASE expressions
  + 3.0 * maximum AST depth
)
```

Сортировка: `score DESC`, затем имя файла. Уровни сложности задаются относительно корпуса: первые 10 запросов — «экстремальная», 11–30 — «высокая», 31–60 — «средняя», остальные — «базовая».

## Разбор одного запроса

Карточка запроса показывает:

- место в рейтинге, балл и уровень;
- число AST-узлов и максимальную глубину;
- CTE и их имена;
- таблицы и количество обращений к ним;
- JOIN и их типы;
- подзапросы и максимальную вложенность;
- `UNION`, `INTERSECT`, `EXCEPT`;
- оконные и агрегатные функции;
- `CASE`, `WHERE`, `HAVING`, `GROUP BY`, `ORDER BY`;
- краткое текстовое объяснение маршрута разбора;
- исходный SQL в раскрываемом блоке.

Ошибки разбора остаются в отчёте отдельными карточками и не теряются.

## Интерфейс

Визуальное направление — техническое редакционное досье: светлая бумажная основа, тёмные чернила, кирпично-красный акцент, выразительная антиквенная типографика и моноширинный SQL. Отчёт содержит:

- сводные показатели корпуса;
- диаграммы суммарных конструкций и уровней сложности;
- поиск по номеру, таблице, CTE, функции и SQL-тексту;
- фильтр по уровню сложности;
- раскрытие всех или только отдельных запросов;
- адаптивную вёрстку для широкого и узкого экрана;
- печатные стили.

## Ограничения

- Не утверждать, что рейтинг предсказывает время или ресурсы выполнения.
- Не использовать выдуманные таблицы и SQL-примеры.
- Не загружать CDN, шрифты, JavaScript или изображения извне.
- Не менять существующий анализ исходного `query_text`/`query_text_template`; генератор отчёта является отдельным режимом CLI.

## Критерии приёмки

1. В отчёте ровно 99 карточек и 99 успешно разобранных файлов.
2. Карточки по умолчанию отсортированы по убыванию балла.
3. Формула сложности видна пользователю.
4. Для каждого запроса доступны метрики разбора и полный SQL.
5. Фильтры и поиск работают без сервера и интернета.
6. Полный набор автоматических тестов проходит.
7. Готовый HTML визуально проверен в браузере на отсутствие переполнений и сломанной структуры.
